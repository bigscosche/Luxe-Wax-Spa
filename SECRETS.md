# Secrets & environment variables

Keys never get hardcoded into committed files. The project uses two
patterns — one for the browser, one for server-side Netlify Functions.

---

## 1. What lives where

| Key | Used by | How it reaches code |
|-----|---------|---------------------|
| `SUPABASE_URL` | admin pages, `review.html` | `window.LUXE_CONFIG` from `/config.local.js` |
| `SUPABASE_ANON_KEY` | admin pages, `review.html` | `window.LUXE_CONFIG` from `/config.local.js` |
| `SUPABASE_SERVICE_KEY` | `netlify/functions/*.js` only | `process.env` — server-side only |
| `SLACK_WEBHOOK_URL` | `netlify/functions/*.js` only | `process.env` — server-side only |
| `TWILIO_*` | `netlify/functions/scheduled-reminders.js` | `process.env` — server-side only |

The **service key** and **Twilio auth token** must never appear in any
`.html` or browser-loaded `.js` file. They live only in Netlify env
vars and are read by Netlify Functions at request time.

The **anon key** is intentionally safe for the browser. Row Level
Security (see `supabase/admin.sql`) enforces access — anon role can
only INSERT into `reviews`.

---

## 2. Files (committed vs. ignored)

| File | Committed? | Purpose |
|------|-----------|---------|
| `.env.example` | committed | Template showing required variable names. |
| `.env` | **ignored** | Your local shell env vars. Optional. |
| `config.example.js` | committed | Template for the browser config. |
| `config.local.js` | **ignored** | Real values. Generated by Netlify in prod, hand-written in dev. |
| `scripts/build-config.sh` | committed | Generates `config.local.js` from env vars at Netlify build time. |

`.gitignore` blocks `.env`, `.env.*` (except `.env.example`), and
`config.local.js`.

---

## 3. Local development

One-time setup:

```bash
# 1. Browser config (admin pages + review page)
cp config.example.js config.local.js
# open config.local.js and paste the real SUPABASE_URL + SUPABASE_ANON_KEY

# 2. Shell env for Netlify CLI (if you'll run netlify dev for functions)
cp .env.example .env
# fill in SUPABASE_URL, SUPABASE_ANON_KEY — add SUPABASE_SERVICE_KEY,
# SLACK_WEBHOOK_URL, TWILIO_* here only if you're testing those functions
```

Run the site:

```bash
# Static only — admin pages + review.html work without functions
npx serve .                       # or any static server

# With functions (lead.js, scheduled-reminders.js)
netlify dev                       # reads .env automatically
```

Before committing, double-check nothing leaked:

```bash
git status                        # config.local.js and .env should NOT appear
git diff --staged | grep -iE 'supabase_(url|anon|service)|sk_|twilio' || echo "clean"
```

## 4. Production (Netlify)

1. **Site settings → Environment variables.** Add:
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_KEY` *(for Netlify Functions — server-only)*
   - `SLACK_WEBHOOK_URL` *(optional)*
   - `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_FROM` *(optional)*

2. **Deploy.** `netlify.toml` runs `bash scripts/build-config.sh`,
   which writes `config.local.js` into the published directory using
   `SUPABASE_URL` and `SUPABASE_ANON_KEY` from the env. The admin
   pages then load `/config.local.js` like any other static asset.

3. The service key / Twilio / Slack keys stay in Netlify env vars only
   — Functions read them via `process.env` at invocation time and
   never touch the browser bundle.

---

## 5. Rotation

If a key is exposed:

1. **Supabase anon key** → Supabase dashboard → Project Settings → API
   → reset anon key. Update the Netlify env var. Redeploy (the build
   regenerates `config.local.js`). Update your local file.

2. **Supabase service key** → same place, reset service key. Update
   Netlify env var only. Functions pick up the new value on next
   invocation. Browser code is unaffected.

3. **If a real key was ever committed**, rewrite history:
   `git filter-repo --path config.local.js --invert-paths` (or use
   GitHub's secret-scanning remediation). Rotate the key regardless —
   cache exists forever.

## 6. Quick reference

```
┌──────────────────────┬──────────────────────────┐
│ Browser needs this   │ window.LUXE_CONFIG.*     │
│                      │ ← /config.local.js       │
│                      │ ← scripts/build-config.sh│
│                      │ ← Netlify env vars       │
├──────────────────────┼──────────────────────────┤
│ Functions need this  │ process.env.*            │
│                      │ ← Netlify env vars       │
│                      │ ← local .env (netlify dev)│
└──────────────────────┴──────────────────────────┘
```
